\subsection{Few-shot CoT 注入}

Few-shot CoT（CoT-FS($k$)）在 CoT-ZS 基础上，于 Stage-1 的 user 段之前前置 $k$ 条 \texttt{(题目, 推理, 代码)} 示例，对应 H3、H6、H10 三组评测。为保证示例与目标题尽量同构，本文采用按强标签检索的方式选池，并在训练侧与评测侧共享同一评分函数。

\paragraph{（1）候选评分与选池}
对每条目标样本 $q$，从 3.3.2 节定义的强标签集合派生 \texttt{strong\_tags}，在候选集合（训练集或评测集扣除 $q$）上按下式打分：
\[
\mathrm{score}(q, q') = 2 \cdot \bigl|\,\mathrm{strong\_tags}(q) \cap \mathrm{strong\_tags}(q')\,\bigr|
                       + \bigl|\,\mathrm{tags}(q) \cap \mathrm{tags}(q')\,\bigr|.
\]
评分在「强标签命中」与「普通标签命中」之间以 2:1 加权，既偏好核心算法范式一致的示例，也兼顾同题型的辅助相似度。按分数降序截取前 $\max(k, 4k)$ 条作为候选池，再在池内做一次随机抽样以避免总是命中同几条高分示例。

\paragraph{（2）训练/评测两侧的随机性差异}
两侧共用同一评分函数与同一强标签集合，只在随机采样环节有别：
\begin{itemize}
    \item \textbf{训练侧}：\texttt{random.sample(pool, k)}，依赖 \texttt{seed=42} 的全局 RNG，每个 epoch 的示例都在统计上来自同一分布；
    \item \textbf{评测侧}：使用题目与样本索引派生的种子 \pth{seed = 2026 + problem_idx * 1000 + sample_idx}，保证同一 \texttt{(题目, 采样编号)} 的示例选择完全可复现，便于跨实验、跨运行做差分归因。
\end{itemize}
这一差异使得训练时 few-shot 上下文分布与评测时保持一致，但评测结果仍可被任何第三方按同一 seed 逐条复核。

\paragraph{（3）示例拼接与预算分配}
$k$ 条示例按「题目 $\to$ 推理 $\to$ 代码块」顺序串联后，整体作为 user 段的前缀，再拼上目标题与 4.4.1 节的指令。三类长度预算在 Stage-1 与 Stage-2 的 prompt 之间共享：
\begin{itemize}
    \item 示例推理保持原文形态（无额外截断），因为其长度已由数据生成阶段控制；
    \item 目标题的 Stage-1 输出仍受 \texttt{MAX\_REASONING\_CHARS} 与 \texttt{MAX\_REASONING\_PROMPT\_TOKENS} 二级截断；
    \item \texttt{VLLM\_MAX\_MODEL\_LEN=8192} 为全部示例 + 目标题 + 两阶段生成预留上下文窗口。
\end{itemize}

\paragraph{（4）训练/推理 $k$ 匹配的强约束}
CoT-FS 组必须满足 4.1 节的合法性约束：当训练侧为 \texttt{LoRA\_cot\_fs(k)} 时，推理侧的 few-shot 数必须同为 $k$，评测脚本对此做显式拒绝（H10 强制 \texttt{--few-shot-k} 取 \texttt{FS\_K}；跨 $k$ 调用直接退出）。该约束确保「训练见 $k$ 例，评测也见 $k$ 例」——这是本文 few-shot 净效应归因的前提，放弃该约束将导致 train/test 上下文长度与示例密度不匹配，从而混淆训练侧与推理侧贡献。
