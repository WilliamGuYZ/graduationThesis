\paragraph{（3）设计依据与一致性约束}
上述两张表的取值遵循以下三条原则，使得 10 组实验之间仅因训练样本构造差异而产生可解释的指标变化：
\begin{itemize}
    \item \textbf{跨适配器一致}：三种 LoRA 共享完全相同的结构与优化器超参，仅 \texttt{train\_cot.py} 多出 \texttt{--num-few-shots} 一个分支参数；
    \item \textbf{与推理侧一致}：训练阶段的 chat template、system 提示、停止 token 均与 4.2.1 节推理 prompt 对齐，避免训练与推理在特殊 token 层面错位；
    \item \textbf{与数据规模匹配}：$3$ epoch $\times$ 等效 batch size 32 在 $\approx$9000 训练样本上约合 $\approx$850 次 optimizer step，足以让 LoRA 收敛而不至于在 CoT 监督目标上过拟合。
\end{itemize}
