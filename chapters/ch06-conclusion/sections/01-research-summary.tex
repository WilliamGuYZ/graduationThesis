% Section 6.1: 研究工作总结

本文以 CodeGeeX4-ALL-9B 为共同基座，在 KodCode 派生语料上构建训练-评测一体化框架，按「训练侧 $\times$ 推理侧」二因素析因展开 H1--H10 共 10 组对照，主要工作如下。

\begin{itemize}
    \item \textbf{统一语料与协议（Ch3）}：构建带 \texttt{thought\_step} 与 \texttt{tags} 的 JSONL 语料（训练 $\approx$9000 / 评测 $\approx$1000 条），10 组实验共享同一 chat template、同一执行器与同一 pass@$k$ 口径；
    \item \textbf{LoRA 三档（Ch4.3）}：\texttt{LoRA\_code} / \texttt{LoRA\_cot\_zs} / \texttt{LoRA\_cot\_fs($k$)} 共享 $r{=}32$、$\alpha{=}64$、7 投影矩阵的同一结构与等效 batch $=32$、3 epoch 调度，差异仅在样本构造；
    \item \textbf{两阶段 CoT（Ch4.4）}：Stage-1 推理 $\to$ 二级截断（\texttt{1200 chars / 256 tokens}）$\to$ Stage-2 代码，训练/评测共享强标签评分与候选池规则，「训练-推理 $k$ 匹配」作为硬约束；
    \item \textbf{析因结论（Ch5）}：% TODO: 待 H1--H10 评测全部完成后，将 5.3 节确定性结论浓缩为 3--4 句话填入此处，与 Ch1.4 贡献 (3) 的析因设问一一对应；附 Logic / Structural / Runtime / Timeout 四类失效归因，超参/seed/长度预算三类消融验证主要结论不被单一扰动主导。
\end{itemize}
