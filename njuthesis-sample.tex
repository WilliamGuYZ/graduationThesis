\begin{abstract}
  近年来，大语言模型（Large Language Models, LLMs）的快速发展推动了软件工程从“人工编码”向“智能生成”加速演进。算法代码生成作为其中的重要研究方向，在提升开发效率、辅助程序设计教学和构建智能编程工具方面具有显著应用价值。然而，现有方法在算法任务中仍面临推理过程不稳定、逻辑规划不足与代码可执行正确性波动等问题，限制了其工程落地能力。基于此，本文围绕大语言模型的算法代码生成开展系统研究，聚焦数据构建、参数高效微调与推理机制优化三个关键环节。

  本文基于 KodCode 数据集构建了面向训练与评测的一体化实验流程，并以专用代码大模型 CodeGeeX4-ALL-9B 为基座，从训练侧与推理侧两个维度展开系统研究。训练侧覆盖 Base、纯代码 LoRA、零样本 CoT 格式 LoRA 与少样本 CoT 格式 LoRA 共四种配置，推理侧覆盖 Direct、Zero-shot CoT 与 Few-shot CoT 三种提示结构；以“训练侧与推理侧若均为 CoT，则示例数量必须一致”为合法性约束，构成共十组有效实验（H1～H10）的二因素析因框架。实验采用统一提示模板与基于自动化单元测试的 pass@k 评测流程，从可执行正确性角度对训练侧主效应、推理侧主效应、二者的交互效应以及少样本边际效应的一致性分别给予量化分析。

  实验结果表明：相较基座模型，LoRA 微调能够有效提升算法代码生成的稳定性与正确性；在此基础上引入 CoT 推理后，模型在复杂问题上的推理连贯性进一步增强；当训练侧与推理侧在示例配置上协同一致时，Few-shot 对解题步骤组织与代码生成鲁棒性的提升最为明显。通过对十组实验主效应与交互项的系统分析，本文在一致协议下验证了“参数高效微调 × 结构化推理引导”在算法代码生成任务中的正交增益与协同条件，并基于端到端打通的 H1～H10 实验链路，为后续智能编程系统优化提供了可复用的实证基础。
\end{abstract}

\begin{abstract*}
  Recent advances in Large Language Models (LLMs) are reshaping software engineering from manual coding toward intelligence-assisted generation. Algorithmic code generation is a key direction in this transition, with practical value for developer productivity, programming education, and intelligent coding systems. However, existing approaches still suffer from unstable reasoning, limited planning ability, and inconsistent executable correctness on algorithmic tasks, which hinders reliable deployment. To address these challenges, this thesis investigates LLM-based algorithmic code generation from three aspects: data construction, parameter-efficient fine-tuning, and reasoning optimization.

  We build an integrated train-evaluate pipeline based on KodCode-style data, and study the code-specialized model CodeGeeX4-ALL-9B along two orthogonal dimensions: a training side covering four configurations (Base, code-only LoRA, zero-shot CoT-format LoRA, and few-shot CoT-format LoRA) and an inference side covering three prompting structures (Direct, Zero-shot CoT, and Few-shot CoT). Imposing the consistency constraint that whenever both sides employ CoT their exemplar counts must match, we obtain a two-factor factorial design with ten valid settings (H1–H10). All settings are evaluated under a unified prompt template and execution-based pass@k unit tests, which allow us to quantify the main effect of the training side, the main effect of the inference side, their interaction, and the consistency of the few-shot marginal effect across training sides.

  Empirical findings show that LoRA improves generation stability and executable correctness over the base model; adding CoT further enhances reasoning coherence on complex problems; and when the training-side and inference-side exemplar configurations are aligned, Few-shot yields the most pronounced gains in step organization and generation robustness. A systematic analysis of main effects and interaction terms across the ten settings validates, under a single protocol, the orthogonal benefits and cooperative conditions of combining parameter-efficient adaptation with structured reasoning guidance for algorithmic code generation, and the end-to-end H1–H10 pipeline established in this work provides a reproducible basis for future optimization of intelligent programming systems.
\end{abstract*}

\begin{notation}[10cm]
  \item[LLM] 大语言模型 (Large Language Model)
  \item[PEFT] 参数高效微调 (Parameter-Efficient Fine-Tuning)
  \item[LoRA] 低秩适配 (Low-Rank Adaptation)
  \item[CoT] 思维链 (Chain-of-Thought)
  \item[ICL] 上下文学习 (In-Context Learning)
  \item[SFT] 有监督微调 (Supervised Fine-Tuning)
  \item[Direct] 单阶段直接生成的提示结构（题目 $\rightarrow$ 代码）
  \item[CoT-ZS] 零样本两阶段思维链推理（Zero-shot CoT）
  \item[CoT-FS($k$)] 含 $k$ 条示例的少样本两阶段思维链推理（Few-shot CoT）
  \item[Base] 不加 LoRA 的原始基座（CodeGeeX4-ALL-9B）
  \item[LoRA\_code] 仅监督 \texttt{solution} 的纯代码 LoRA 适配器
  \item[LoRA\_cot\_zs] 监督 \texttt{thought\_step + solution} 且训练侧无示例的 CoT LoRA 适配器
  \item[LoRA\_cot\_fs($k$)] 同上，但训练侧含 $k$ 条 few-shot 示例
  \item[H1--H10] 训练侧 $\times$ 推理侧二因素析因设计下的十组有效实验
  \item[pass@$k$] 每题采样 $N$ 次中至少一次通过单测的题目占比的无偏估计
  \item[$r$] LoRA 低秩分解秩，本文取 32
  \item[$\alpha$] LoRA 缩放系数，本文取 64
  \item[$k$] Few-shot 示例条数，本文固定 $k{=}2$
  \item[$N$] 每题采样数，本文取 10，支撑 pass@1/5/10 估计
  \item[DiD] 差分差分量 (Difference-in-Differences)，用于检验交互效应
\end{notation}

\chapter{Prompt 模板字面全文}
\label{appendix:prompts}

本附录给出第四章三类训练侧与三类推理侧 Prompt 的逐字模板，便于第三方按本文协议复现 H1--H10 的实验结果。占位符 \texttt{\{func\}}、\texttt{\{params\}}、\texttt{\{question\}}、\texttt{\{thought\}}、\texttt{\{code\}} 的来源与第四章 4.1 节（1a）一致。

\section{Direct 推理侧（H1 / H4 / H7 / H9）}
单阶段、一次 \texttt{generate}，对应 \texttt{evaluation/eval\_base\_passk.py} 与 \texttt{eval\_lora\_passk.py}。
\begin{itemize}
    \item \textbf{System}：
\begin{quote}\ttfamily\footnotesize
You are an expert Python programmer. Write correct Python code to solve the given problem.
\end{quote}
    \item \textbf{User}：
\begin{quote}\ttfamily\footnotesize
Implement the function `\{func\}' that takes \{params\} to solve:\\[0.3em]
\{question\}\\[0.3em]
Only output the Python code, no explanation.
\end{quote}
\end{itemize}

\section{CoT-ZS 推理侧（H2 / H5 / H8）}
两阶段、两次 \texttt{generate}，对应 \texttt{evaluation/eval\_lora\_cot\_passk.py --few-shot-k 0}。

\paragraph{Stage-1（推理生成）}
\begin{itemize}
    \item \textbf{System}：
\begin{quote}\ttfamily\footnotesize
You are an expert algorithm teacher. Generate step-by-step reasoning.
\end{quote}
    \item \textbf{User}：
\begin{quote}\ttfamily\footnotesize
Implement the function `\{func\}' that takes \{params\} to solve:\\[0.3em]
\{question\}\\[0.3em]
Think step by step, then output the Python code.
\end{quote}
\end{itemize}

\paragraph{Stage-2（代码生成）}
Stage-1 输出经 \texttt{MAX\_REASONING\_CHARS=1200} 与 \texttt{MAX\_REASONING\_PROMPT\_TOKENS=256} 二级截断后写入 Stage-2 上下文：
\begin{itemize}
    \item \textbf{System}：
\begin{quote}\ttfamily\footnotesize
You are an expert Python programmer. Based on the reasoning, write the code.
\end{quote}
    \item \textbf{User}：
\begin{quote}\ttfamily\footnotesize
Reasoning:\\[0.2em]
\{truncated\_stage1\_text\}\\[0.5em]
Now implement the function `\{func\}' that takes \{params\} to solve:\\[0.3em]
\{question\}\\[0.3em]
Write only the Python code in \verb|```python| block.
\end{quote}
\end{itemize}

\section{CoT-FS($k$) 推理侧（H3 / H6 / H10）}
在 CoT-ZS 两阶段 system 各自之后插入 $k$ 组示例，再接目标题；两阶段共享同一 $k$ 组、相同顺序。每组示例形如：
\begin{itemize}
    \item \textbf{User（示例题，CoT 版指令）}：
\begin{quote}\ttfamily\footnotesize
Implement the function `\{shot\_func\}' that takes \{shot\_params\} to solve:\\[0.3em]
\{shot\_question\}\\[0.3em]
Think step by step, then output the Python code.
\end{quote}
    \item \textbf{Assistant（示例 \texttt{thought\_step + fenced solution}）}：
\begin{quote}\ttfamily\footnotesize
\{shot\_thought\}\\[0.5em]
\verb|```python|\\
\{shot\_code\}\\
\verb|```|
\end{quote}
\end{itemize}
示例选择：按 $\mathrm{score}(q, q') = 2 \cdot |\mathrm{strong\_tags}(q) \cap \mathrm{strong\_tags}(q')| + |\mathrm{tags}(q) \cap \mathrm{tags}(q')|$ 评分取 top $\max(k, 4k)$，再以 \texttt{seed = 2026 + problem\_idx \(\times\) 1000 + sample\_idx} 抽样 $k$ 条。

\section{LoRA 训练侧 Chat 模板}

\paragraph{LoRA\_code（H4 训练）}
单条样本为 system + user + assistant 共 3 条，user/system 与 Direct 推理逐字同构；assistant 为可执行 Python 正文（\textbf{无} Markdown 围栏）。Loss 仅落在 assistant 段。

\paragraph{LoRA\_cot\_zs（H7 / H8 训练）}
\begin{itemize}
    \item \textbf{System}：
\begin{quote}\ttfamily\footnotesize
You are an expert Python programmer. Before writing code, think through\\
the problem step by step. Format your response as numbered reasoning\\
steps followed by the Python solution.
\end{quote}
    \item \textbf{User}：与 CoT-ZS Stage-1 相同；
    \item \textbf{Assistant}：\texttt{\{thought\}} + 空行 + \verb|```python|\,\texttt{\{code\}}\,\verb|```|。
\end{itemize}

\paragraph{LoRA\_cot\_fs($k$)（H9 / H10 训练）}
在 \texttt{LoRA\_cot\_zs} 的 system 之后拼接 $k$ 组示范（与推理侧 CoT-FS 中 user/assistant 字面同构），再接当前题 user 与 assistant；前 $k$ 组示例 labels 全部为 \texttt{-100}，loss 仅落在当前题 assistant 段。

\chapter{评测脚本关键代码段}
\label{appendix:codes}

本附录摘录第 4 章流程中三段独立可读的核心实现，便于读者按字面复现评测口径。完整脚本见 \texttt{ati-codegen/evaluation/}。

\section{pass@k 无偏估计}
对应 \cite{chen2021evaluating} 提出的无偏估计，三份评测脚本（\texttt{eval\_base\_passk.py} / \texttt{eval\_lora\_passk.py} / \texttt{eval\_lora\_cot\_passk.py}）实现完全一致：
\begin{quote}\ttfamily\footnotesize
def pass\_at\_k(n: int, c: int, k: int) -> float:\\
\hspace*{2em}if n < k:\\
\hspace*{4em}return float("nan")\\
\hspace*{2em}if n - c < k:\\
\hspace*{4em}return 1.0\\
\hspace*{2em}return 1.0 - comb(n - c, k) / comb(n, k)
\end{quote}
其中 $n$ 为单题采样数（本文取 \texttt{NUM\_SAMPLES=10}），$c$ 为通过数；公式等价于 $1 - \binom{n-c}{k} / \binom{n}{k}$。

\section{代码抽取（Direct 取首段、CoT 取末段）}
4.2.2 节 Direct 路径与 4.4.1 节 CoT 路径的代码抽取规则在脚本中以两份 \texttt{extract\_code} 实现，差异仅在 \texttt{[0]} 与 \texttt{[-1]}：
\begin{quote}\ttfamily\footnotesize
\# eval\_base\_passk.py / eval\_lora\_passk.py（首段，Direct）\\
def extract\_code(text: str) -> str:\\
\hspace*{2em}code\_blocks = re.findall(r"```python(.*?)```", text, re.S)\\
\hspace*{2em}if not code\_blocks:\\
\hspace*{4em}code\_blocks = re.findall(r"```(.*?)```", text, re.S)\\
\hspace*{2em}if code\_blocks:\\
\hspace*{4em}return code\_blocks[0].strip()\\
\hspace*{2em}return text.strip()\\[0.4em]
\# eval\_lora\_cot\_passk.py（末段，CoT-FS 时绕开 few-shot 回放）\\
def extract\_code(text: str) -> str:\\
\hspace*{2em}\dots\\
\hspace*{2em}return code\_blocks[-1].strip()
\end{quote}

\section{Stage-1 输出截断}
对应 4.4.1 节（1）「避免推理阶段偷跑代码污染第二阶段上下文」：
\begin{quote}\ttfamily\footnotesize
def \_clip\_reasoning\_before\_code(text: str) -> str:\\
\hspace*{2em}idx\_py  = text.find("```python")\\
\hspace*{2em}idx\_any = text.find("```")\\
\hspace*{2em}cut\_idx = min(i for i in [idx\_py, idx\_any] if i >= 0) \\
\hspace*{4em}if (idx\_py >= 0 or idx\_any >= 0) else -1\\
\hspace*{2em}if cut\_idx >= 0:\\
\hspace*{4em}return text[:cut\_idx].rstrip()\\
\hspace*{2em}return text.strip()
\end{quote}
此处取 \texttt{```python} 与 \texttt{```} 两个起始位置中的较小者，意味着任意三反引号围栏（含 \texttt{```bash} 等其他语言标记或无标记块）均会触发截断，与 4.4.1 节（1）「输出清洗」一致。
